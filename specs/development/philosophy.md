# Development Philosophy

- Optimize for **speed of the development loop**, not avoidance of all failures.
- Feel failure domains - recover fast rather than gate everything.
- If a ceremony or gate slows development down, engineer away the reason it exists.
- Humans steer, agents execute. When an agent struggles, ask: **"what capability is missing?"** - then build it.
- No manually-written code is the aspiration. Humans work at the design/spec/feedback layer.
